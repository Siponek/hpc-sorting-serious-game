\chapter*{Abstract}
\addcontentsline{toc}{chapter}{Abstract}

High-Performance Computing (HPC) and parallel programming remain challenging topics in computer science education, often requiring students to grasp abstract concepts without tangible, interactive experiences. Serious games provide pathways for learners to develop intuition about concepts that are new to them, acting as engaging ice-breaker activities that lead to deeper questions and discussions, allowing instructors to uncover misconceptions and clarify understanding.

This thesis presents the design, implementation, and evaluation of an innovative educational serious game that teaches fundamental parallel computing paradigms—specifically OpenMP (shared memory).
%and MPI (distributed memory)—through an interactive card-sorting mechanic on mobile devices.
The game transforms abstract parallel computing concepts into concrete, manipulable interactions: players sort cards collaboratively in multiplayer mode
%(simulating MPI message passing) or individually 
with private buffer zones (simulating OpenMP shared memory with thread-local storage).

The goal is to enable students to cooperate via a Web-based interface to sort a set of cards in the shortest time within given conditions, thereby learning basic HPC concepts such as speedup, scalability, and communication overheads, as well as parallel programming paradigms including multithreading and message passing.

By implementing this game using the Godot Engine and GDScript with web export capabilities, this work addresses both pedagogical and technical challenges inherent in creating educational multiplayer games for web browsers. The development process required identifying a proper platform supporting extensible development with networking capabilities, followed by developing a working prototype.

Key contributions include: (1) a novel pedagogical mapping between card-sorting mechanics and HPC paradigms based on physical classroom experiments, (2) implementation of robust multiplayer state synchronization for educational purposes using the GDSync framework, (3) solutions to responsive UI/UX challenges for displaying and manipulating numerous interactive elements (50+ cards) across different screen sizes in web browsers, (4) comprehensive documentation of technical challenges encountered and solutions implemented, particularly regarding the GDSync framework and multiplayer synchronization issues, and (5) an open-source, extensible platform for future HPC education research suitable for publication in conferences and scientific journals.

\textbf{Keywords:} Serious Games, High-Performance Computing, Parallel Computing Education, OpenMP,

Mobile Game Development, Multiplayer Synchronization, Godot Engine, GDScript, Card Sorting, Educational Technology
